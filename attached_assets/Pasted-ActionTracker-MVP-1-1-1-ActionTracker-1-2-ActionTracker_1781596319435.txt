ActionTracker 要件定義書（MVP版） 
1. システム概要 
1.1 システム名 
ActionTracker 
1.2 システム概要 
ActionTrackerは、ユーザーのPC上での行動履歴を収集・分析し、単なる利用時間の記録では
なく「どのような行動の流れで時間を使ったのか」を可視化する行動理解プラットフォームである。 
ユーザーが自身の行動パターンや集中を妨げる要因を客観的に把握し、自己理解や生産性向
上につなげることを目的とする。 
1.3 解決する課題 
既存のスクリーンタイムツールでは以下しか把握できない。 
● アプリ利用時間 
● サービス利用時間 
しかし、 
● なぜ集中が途切れたのか 
● どのような経路で脱線したのか 
● どのような行動パターンを繰り返しているのか 
は把握できない。 
ActionTrackerは行動履歴をストーリーとして再構築することで、ユーザーの自己理解を支援す
る。 
2. 対象ユーザー 
2.1 メインターゲット 
● 学生 
● プログラミング学習者 
● 個人開発者 
● リモートワーカー 
● フリーランス 
2.2 ユーザー課題 
● 集中したつもりでも実際の行動が分からない 
● どこで集中が途切れたか把握できない 
● 行動パターンを客観的に確認できない 
● 時間の使い方を改善したい

4. 機能要件 
F-001 行動イベント収集 
概要 
PC上の行動イベントを収集する。 
入力 
● アクティブウィンドウ変更 
● マウス操作 
● キーボード操作 
● アイドル状態変化 
出力 
イベントログ 
取得項目 
● タイムスタンプ 
● アプリ名 
● サービス名 
● カテゴリ 
● ウィンドウタイトル 
● イベント種別 
保存方式 
イベント駆動方式 
状態変化時のみ記録する。 
F-002 リアルタイムダッシュボード 
表示内容 
● 現在利用中アプリ 
● 利用開始時刻 
● 継続時間 
● 本日の利用時間 
● 本日の切替回数 
更新間隔 
10秒 
F-003 タイムライン表示 
概要 
1日の行動履歴を時系列で表示する。 
表示形式 
ガントチャート 
表示項目 
● 開始時刻 
● 終了時刻 
● アプリ名 
● サービス名 
● カテゴリ 
● 利用時間 
F-004 サービス遷移分析 
概要 
サービス間およびカテゴリ間の遷移を分析する。 
表示内容 
● 遷移回数ランキング 
● 頻出遷移パターン 
例 
VSCode → GitHub 
GitHub → YouTube 
YouTube → VSCode 
F-005 行動ストーリー生成 
概要 
行動履歴を人が理解しやすい文章へ変換する。 
出力例 
14:02 
VSCodeで開発を開始 
14:18 
GitHubでエラー調査 
14:24 
YouTubeで関連動画を視聴 
14:43 
VSCodeへ復帰 
推定離脱時間：19分 
F-006 インサイト生成 
概要 
行動パターンから傾向を抽出する。 
出力例 
● 調査開始後に動画サイトへ遷移する傾向があります 
● 14時台にSNS利用頻度が増加しています 
● 集中時間は午前中が最も長い傾向があります 
F-007 行動カテゴリ分析 
カテゴリ例 
● 開発 
● 学習 
● 娯楽 
● SNS 
● コミュニケーション 
出力例 
開発：4時間12分 
学習：2時間04分 
SNS：47分 
F-008 プライバシーモード 
対象 
● 銀行サイト 
● 証券サイト 
● メール 
● パスワードマネージャー 
動作 
対象サービスは以下をマスクする。 
● ウィンドウタイトル 
● サービス名 
F-009 サービス正規化 
概要 
取得したウィンドウタイトルを解析し、人間が理解しやすいサービス名へ変換する。 
入力例 
YouTube - Microsoft Edge 
ChatGPT - Microsoft Edge 
Visual Studio Code 
出力例 
YouTube 
ChatGPT 
VS Code 
目的 
ブラウザ実行ファイル名（msedge.exe、chrome.exe等）だけでは判別できないユーザー行動を、
サービス単位で分析可能にする。
